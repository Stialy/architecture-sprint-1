## Задание 1

### Уровень 1. Проектирование

Был выбран подход __Web Module Federation__
* Требований к использованию различных стеков нет;
* В проекте используется React и команда имеет опыт работы на нем;
* Имеется общие компоненты:
	* Стили
	* Шрифты;
* Обеспечивает независимость развертывания;
* Загрузка кода по требованию (__lazy loading__);
* Позволяет использовать разные версии библиотек (Сейчас не актуально, но с развитием проекта может дать дополнительные плюсы, когда обновление одного модуля не будет тормозиться иными версиями библиотек в других);


### Уровень 2. Планирование изменений
Разделение на модули

Для разделения на модули использую стратегию __Вертикальной нарезки__
* Авторизация - регистрация новых пользователей и авторизация существующих;
* Профиль пользователя;
* Фотограции и лайки;

Лайки не стал выделять в отдельный микрофронтенд, функционал сильно связан с фотографиями и прост.
При развитии требований к Лайкам расмотрел бы вопрос о выделении функционала в отдельный микрофронтенд

* Хост - для реализации подхода __Web Module Federation__
